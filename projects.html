SRCF Control Panel

When the SRCF first started, users had to email us for all requests, e.g. resetting their passwords and setting up a custom domain. We then acted on these requests manually by running appropriate scripts. At the end of 2017, we successfully launched a control panel to let users manage their accounts directly without having to go through the sysadmins. The frontend part of the code is on GitHub.

While the control panel is already in use, there is still development work to do:

  • We still run the scripts by hand every now and then. We are currently working on srcflib, which will act as a backend for both the control panel and the manual scripts to avoid duplicating to logic.

  • There are some functionality that are yet to be integrated with the control panel, such as getting letsencrypt certificates for custom domains. Implementing these would be a good way to learn how the control panel works.

Ansible setup

Historically, SRCF machines are set up by hand. Unfortunately, this often means the documentation for the setup only exists in one personâ€™s head. It is also cumbersome to reinstall machines when we have to. We have automated some of this via the srcf-server package that does the setup common to all our VMs, but individual setups are largely poorly documented.

We are now moving towards using Ansible to set up our VMs. This lets us write config files that describe how a system should be set up, instead of running commands by hand. These can be version controlled and passed on to future sysadmins. Moreover, this allows us to test our configuration and changes on a staging machine before deploying.

At the moment, our user service server downpour (which hosts, e.g. Mattermost) has all its services set up via ansible, and we hope to expand this to more and more of our machines.
